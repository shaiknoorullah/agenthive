# agenthive libp2p Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. This plan is the source of truth for an automated Workflow that dispatches a fresh subagent per package, runs two-stage review (spec compliance → code quality), integrates, verifies, pushes, watches CI, fixes failures, and merges.

**Goal:** Ship a libp2p-based agenthive daemon — transport, identity, discovery, hooks, dispatch, daemon, CLI — that builds, passes `go test -race`, and merges to `main` via a green CI PR. No surfaces beyond a log surface. No TUI. Future pushes own those.

**Architecture:** Single Go binary. All wire concerns (transport, identity, peer discovery, NAT traversal, encryption, multiplexing) delegated to go-libp2p. CRDT state layer (`internal/crdt/`) is untouched — it's the payload. The action-gate file-queue mechanism from the original hooks plan is preserved (it's local-device-only and orthogonal to transport). Cobra drives the CLI.

**Tech Stack:** Go 1.22, `github.com/libp2p/go-libp2p`, `github.com/libp2p/go-libp2p-pubsub`, `github.com/spf13/cobra`, `github.com/stretchr/testify`, `pgregory.net/rapid`.

**Authority:** This plan implements `docs/rfcs/adopt-libp2p.md` (which supersedes `docs/superpowers/plans/2026-03-26-transport.md` and parts of the core-daemon, hooks, and dispatch plans).

**Commit policy (non-negotiable):** Conventional Commits. No `Co-Authored-By: Claude`, no "Generated with Claude Code" footer, no AI attribution anywhere in commits or PR body. Every subagent that commits must be told this explicitly.

```

Layer 2  Mid      :   transport, dispatch/log_surface, hooks/gate
Layer 3  Outer    :   discovery, daemon/socket
Layer 4  Top      :   daemon (Run loop wires transport+discovery+protocols+dispatch+hooks)
Layer 5  CLI      :   cmd/agenthive (cobra subcommands + main.go)
Layer 6  Verify   :   go build / test -race / vet / fuzz / lint
Layer 7  Ship     :   push, open PR, watch CI, fix-loop, merge
```

Parallelism is at the package level. Inside a layer, packages are independent and run concurrently. Between layers, there is a barrier — all packages in layer N must succeed before layer N+1 starts.

**Files:**

**Public surface:**

```go
package protocols

const (
    ProtoActionRequest  = "/agenthive/action/request/1"
    ProtoActionResponse = "/agenthive/action/response/1"
    ProtoNotification   = "/agenthive/notification/1"
    ProtoPeerAnnounce   = "/agenthive/peer/announce/1"

    TopicState = "/agenthive/state/v1"
)

// All messages JSON-serialized; size-prefixed framing on streams.

type ActionRequest struct {
    ActionID   string    `json:"action_id"`
    SessionID  string    `json:"session_id"`
    ToolUseID  string    `json:"tool_use_id"`
    ToolName   string    `json:"tool_name"`
    ToolInput  string    `json:"tool_input"`
    Project    string    `json:"project,omitempty"`
    CWD        string    `json:"cwd,omitempty"`
    Timestamp  time.Time `json:"ts"`
    ExpiresAt  time.Time `json:"expires_at"`
}

type ActionResponse struct {
    ActionID   string `json:"action_id"`
    Decision   string `json:"decision"` // "allow" | "deny" | "allow-always"
    DecidedBy  string `json:"decided_by"` // peer ID that decided
}

type Notification struct {
    SessionID string    `json:"session_id"`
    Source    string    `json:"source"`    // "claude-code" | "codex" | ...
    Project   string    `json:"project,omitempty"`
    Priority  string    `json:"priority"`  // "info" | "warning" | "critical"
    Message   string    `json:"message"`
    Timestamp time.Time `json:"ts"`
}

type PeerAnnounce struct {
    PeerID    string   `json:"peer_id"`
    Multiaddrs []string `json:"multiaddrs"`
    Timestamp time.Time `json:"ts"`
}

type StateDelta struct {
    From  string `json:"from"`  // sender peer ID
    Peers []byte `json:"peers"` // marshaled LWWMap[PeerInfo]
    Routes []byte `json:"routes"`
    Config []byte `json:"config"`
}

// Helpers
func WriteFramed(w io.Writer, v any) error  // 4-byte length prefix + JSON body
func ReadFramed(r io.Reader, v any) error
```

**Tests:** round-trip every message type through `WriteFramed`/`ReadFramed`; bound checks (reject framing > 16 MiB).

package hooks

type Gate struct {

}

**Files:**

**Public surface:**

```go
package daemon

type Config struct {
    ConfigDir   string
    Identity    crypto.PrivKey
    ListenAddrs []string
    LogPath     string
    SocketPath  string
}

type Daemon struct {
    cfg       Config
    host      host.Host
    state     *crdt.StateStore
    gossipsub *pubsub.PubSub
    topic     *pubsub.Topic
    sub       *pubsub.Subscription
    dispatcher *dispatch.Dispatcher
    gate      *hooks.Gate
    socket    *SocketServer
}

func New(cfg Config) (*Daemon, error)
func (d *Daemon) Run(ctx context.Context) error
func (d *Daemon) Stop() error
```

`Run` workflow:
1. Build the libp2p Host via `transport.New(ctx, cfg)`.
2. Build CRDT StateStore; load from `<configDir>/state.json` if present.
3. Build GossipSub: `pubsub.NewGossipSub(ctx, host)`, `Join(TopicState)`, `Subscribe()`.
4. Register stream handlers on the Host: `ProtoActionRequest`, `ProtoActionResponse`, `ProtoNotification`, `ProtoPeerAnnounce`. Decode framed JSON, route appropriately.
5. Start mDNS via `discovery.StartMDNS` with a callback that adds discovered peers to the StateStore.
6. Start the SocketServer goroutine.
7. Start a goroutine that subscribes to the GossipSub topic and applies `StateDelta` messages to the StateStore.
8. Start a goroutine that periodically publishes deltas of the StateStore via the topic (debounced 200ms, max once per 2s when active).
9. Connect to all peers in `StateStore.ListPeers()`.
10. Block on `ctx.Done()`. On cancel, close everything in reverse order. Persist state to `state.json`.

**Tests:**
- `TestDaemon_StartStop` — boots and shuts down cleanly
- `TestDaemon_TwoPeersExchangeState` — two daemons on different ephemeral ports, connect one to the other via multiaddr; modify state on one, assert it converges on the other within 5s
- All tests use `t.TempDir()`, `defer Stop()`, are `-race`-clean

### Task L0.A — Fix HLC zero-time monotonic bug

**Files:** modify `internal/crdt/hlc.go`

The current `HLC.Now()` returns identical timestamps for two consecutive calls when wall time is zero (test seeds `wallFn` to a constant zero). The `.fail` corpus under `testdata/rapid/` shows this.

**Fix shape:** add an `initialized bool` field to `HLC`. In `Now()`, if `!initialized` OR if `newWall == h.last.Wall && h.last.Counter == 0`, force the counter to advance by setting `h.last.Counter = max(h.last.Counter, 0) + 1`. The robust invariant: every call to `Now()` must return a Timestamp strictly After the previous return value.

**Verification:**

```bash
rm -rf internal/crdt/testdata/rapid
go test -race -count=1 -rapid.checks=1000 ./internal/crdt/...
ls internal/crdt/testdata/rapid 2>/dev/null && exit 1 || echo "clean"
```

**Commit message:**
```
fix(crdt): HLC.Now monotonic guarantee from cold-start zero wall

The previous implementation could return two equal timestamps when the
wall clock returned zero (e.g., in tests using a constant clock), because
h.last was initialized to Timestamp{} and the first Now() call hit the
"wall ahead, reset counter" branch with no actual advance.

Track an initialized flag and ensure every Now() returns a strictly After
timestamp. Property test (TestHLC_Property_Monotonic) confirms.
```
