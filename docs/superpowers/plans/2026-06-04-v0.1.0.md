# agenthive v0.1.0 Implementation Plan — TUI + tmux + Release

> **For agentic workers:** REQUIRED SUB-SKILL: `superpowers:subagent-driven-development`. This plan is executed by a Workflow that dispatches a fresh subagent per package, integrates, verifies, pushes, watches CI, fixes failures, merges, then tags `v0.1.0` and lets GoReleaser publish the binaries.

**Goal:** ship agenthive v0.1.0 — a fully functional, tested, released binary with a working TUI, working tmux surface + plugin, working desktop notification surface, a routing matcher that uses CRDT routes to fan out, AutoRelay correctly fed by the CRDT peer set, and GoReleaser-published artifacts.

**Architecture:** v0.1.0 finishes the surface layer above the libp2p substrate that landed in #11. CRDT data layer, transport, identity, discovery, protocols, hooks, daemon are already shipped. This push adds: TUI (bubbletea), tmux surface adapter, desktop surface adapter, route matcher, peer-source closure that closes the no-op gap in the libp2p RFC, GoReleaser config, release workflow.

**Tech Stack additions:**
- `github.com/charmbracelet/bubbletea` — TUI runtime
- `github.com/charmbracelet/lipgloss` — TUI styling

- `github.com/muesli/termenv` — color profile detection (set to ASCII in tests for determinism)

**Authority:** This plan extends `docs/rfcs/adopt-libp2p.md` (still binding); finishes the items listed as deferred in its §1 status table and the README's status matrix.

**Commit policy (non-negotiable):** Conventional Commits. **No `Co-Authored-By: Claude`, no "Generated with Claude Code" footer, no AI attribution anywhere in commits or PR body.** Every subagent receives this rule via the workflow preamble.

## Package contracts

### Task L2.A — `internal/router/`

**Files:** `internal/router/router.go`, `internal/router/router_test.go`

**Surface:**

```go
package router

type Matcher struct {
    store *crdt.StateStore
    self  peer.ID
}

func NewMatcher(store *crdt.StateStore, self peer.ID) *Matcher

// Targets returns the deduplicated set of target peer IDs (excluding self)
// for a notification, by walking all routes and unioning the targets whose
// RouteMatch selector accepts the notification's metadata.
//
// Selector fields are treated as conjunction: ALL set fields must equal the
// notification's corresponding fields. Unset selector fields are wildcards.
// Targets containing the literal string "ALL" expand to every peer in
// store.ListPeers() except self.
func (m *Matcher) Targets(notif protocols.Notification) []peer.ID
```

**Tests:**
- Empty store → empty target list
- Single matching route → that target
- Multiple matching routes → union of targets
- "ALL" expansion → every peer except self
- Wildcard selector → matches everything
- Project mismatch → no match
- Priority match + project wildcard → match

### Task L2.B — `internal/transport/host.go` peer-source closure update

**Files:** modify `internal/transport/host.go`; add tests if not already present.

The existing `EnableAutoRelayWithPeerSource` callback should be updated to surface the StateStore's peer set as `peer.AddrInfo` channels. The `transport.New` signature gets one extra opt-in parameter:

```go
type Config struct {
    Identity     crypto.PrivKey
    ListenAddrs  []string
    EnableMDNS   bool
    PeerSource   func(ctx context.Context, num int) <-chan peer.AddrInfo  // NEW
}
```

If nil, the existing no-op closure stays. The daemon now passes a closure that calls `store.ListPeers()`, converts each to `peer.AddrInfo`, and emits up to `num` of them.

**Tests:** verify `MultiaddrsFor` still returns sane multiaddrs; verify the host can be constructed with and without `PeerSource`.

**Surface:**

// the tmux status line renders without forking shells.
type TmuxSurface struct {
    exec CmdExecutor  // mockable subprocess runner
}

For each notification, the surface writes (server-globally for v0.1.0; per-pane refinement deferred):

```
tmux set-option -g @notif-msg     <message>
tmux set-option -g @notif-project <project>
tmux set-option -g @notif-source  <source>
tmux set-option -g @notif-time    <ISO-8601 stamp>
tmux set-option -g @notif-priority <priority>
```

`DispatchAction` writes an action prompt similarly to `@notif-action-id` / `@notif-action-tool`.

**Tests:**
- Mock CmdExecutor records the exact arg lists
- Verify shell-safe escaping of message body
- `Close()` clears the options

### Task L2.D — `internal/dispatch/desktop_surface.go`

**Files:** `internal/dispatch/desktop_surface.go`, `internal/dispatch/desktop_surface_test.go`

**Surface:**

```go
package dispatch

type DesktopSurface struct {
    exec CmdExecutor
    os   string  // "linux" or "darwin"
}

func NewDesktopSurface(exec CmdExecutor) *DesktopSurface
// implements Surface
```

Dispatches via:
- Linux: `notify-send --urgency=<low|normal|critical> --app-name=agenthive "<title>" "<body>"`
- macOS: `osascript -e 'display notification "<body>" with title "<title>" subtitle "<source>"'`

Detects OS via `runtime.GOOS`. Other OS → no-op error from `Name() == "desktop:unsupported"`.

**Tests:**
- Linux path: mock exec, verify args
- Darwin path: mock exec, verify args
- Other OS: no error, no exec call
- Priority → urgency mapping

### Task L2.E — `cmd/agenthive/cmd_routes.go`

**Files:** `cmd/agenthive/cmd_routes.go`, `cmd/agenthive/cmd_routes_test.go`

```
agenthive routes add <id> <selector> <target1,target2,...>
agenthive routes list
agenthive routes del <id>
```

Selector grammar v0.1.0 (minimal):

```
agent:<name>        project:<name>      session:<id>
window:<idx>        pane:<id>           source:<name>
priority:<level>    default             *
```

Multiple selector clauses separated by `,` (AND). e.g. `agenthive routes add critical-phone "priority:critical,source:claude-code" "phone,laptop"`.

Tests: parse selector, write via local StateStore, round-trip via `list`, delete via `del`.

### Task L3 — `internal/tui/` (bubbletea)

**Files:**

```
internal/tui/
├── styles.go          # TokyoNight palette via lipgloss; tests just construct + assert non-nil
├── styles_test.go
├── app.go             # root bubbletea Model with tab routing
├── app_test.go        # uses teatest, golden files
├── peers.go           # PeersModel: lists peers, cursor, status icons
├── peers_test.go
├── routes.go          # RoutesModel: lists routes; add/delete via key bindings
├── routes_test.go
├── actions.go         # ActionsModel: lists pending actions; approve (y) / deny (n)
├── actions_test.go
├── logs.go            # LogsModel: list events; level filter (1=info, 2=warn, 3=crit)
├── logs_test.go
└── testdata/          # golden files
```

Common contract for every tab Model:

```go
type Model struct {
    styles Styles
    width  int
    height int
    cursor int
    // tab-specific fields
}

func New(styles Styles) Model
func (m Model) Init() tea.Cmd
func (m Model) Update(msg tea.Msg) (Model, tea.Cmd)
func (m Model) View() string
```

The root `App` switches between tabs on key `p` / `r` / `a` / `l`; quits on `q` or `ctrl+c`; updates child models with a typed `tea.Msg` (`PeersUpdateMsg`, `RoutesUpdateMsg`, etc.) when the daemon delivers new data.

Tests use `teatest.NewTestModel`, send key sequences, assert on `tm.Output()` golden-compared with `teatest.RequireEqualOutput`. All tests force `termenv.Ascii` color profile (no truecolor in CI) by setting `lipgloss.SetColorProfile(termenv.Ascii)` in TestMain.

Each tab Model gets at least 4 tests:
- Initial render → golden
- Cursor navigation (j/k or arrows) → golden after move
- Window resize → render still bounded
- Tab-specific action (approve action, delete route, filter logs)

### Task L4 — `internal/tui/app.go` (the integration)

App owns child models, dispatches Update via type switch, renders the active tab plus a footer with key hints. teatest integration test boots the App, asserts initial render is the Peers tab, presses `r` and asserts it's now Routes tab.

### Task L5.A — `cmd/agenthive/cmd_tui.go`

`agenthive tui` connects to the local daemon's Unix socket (default `<configDir>/agenthive.sock`), receives a snapshot of peers + routes + pending actions + recent logs, launches the bubbletea `App`, periodically polls (every 2s) for updates and dispatches them as `tea.Msg`.

If the daemon socket is unreachable, exit 1 with a clear error message: "agenthive daemon is not running. Start it with: agenthive start".

Tests: in-process happy path using a stub daemon-socket server.

### Task L5.B — `internal/daemon/daemon.go` wires the new pieces

Modify the Run loop to:

1. Build `router.NewMatcher(store, host.ID())`.
2. Construct `dispatch.NewTmuxSurface(exec.New())` if tmux is present in `$PATH`; add to dispatcher.
3. Construct `dispatch.NewDesktopSurface(exec.New())`; add to dispatcher.
4. When dispatching a notification, ask the matcher for target peers first; route the libp2p stream only to those peers (rather than every peer); always also call `dispatcher.Dispatch(ctx, notif)` for local surfaces.
5. Pass a real peer-source closure to `transport.New` that surfaces `store.ListPeers()` as `peer.AddrInfo`.
6. Expose a query API on the Unix socket for the TUI: `list_peers`, `list_routes`, `list_actions`, `list_logs`.

Daemon tests: assert that the new query API responds; assert that a notification with a matching route reaches only the matched peers (use a 3-host in-memory mock topology).

### Task L5.C — `tmux/agenthive.tmux` + helper scripts

**Files:**
- `tmux/agenthive.tmux`
- `tmux/scripts/notification-clear.sh`
- `tmux/README.md` (install instructions)

`tmux/agenthive.tmux` is a TPM-compatible bash script. It must:
- Append `#{?@notif-msg,#[fg=yellow] #{@notif-msg}#[default],}` to the user's `status-right` (idempotent — detect previous install via a sentinel comment).
- Install a `pane-focus-in` hook that runs the clear script: `set-hook -g pane-focus-in 'run-shell "tmux set -gu @notif-msg; tmux set -gu @notif-project; tmux set -gu @notif-source; tmux set -gu @notif-time; tmux set -gu @notif-priority"'`.
- Define `@agenthive-installed` sentinel option so re-sourcing is safe.

Tests for these scripts:
- `bash -n` syntax check on every shell file
- grep assertions: the plugin sets the expected hook and format string
- Optional integration with `tmux -V` available: spawn a server, run the plugin, set an option, verify it appears in the rendered status

CI gating: real-tmux integration tests `t.Skip` if `tmux -V` not in PATH; the unit tests run everywhere.
